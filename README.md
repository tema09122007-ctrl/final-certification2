## Описание программы

GitHub User Finder — это графическое приложение на Python (Tkinter) для поиска пользователей GitHub по логину, просмотра базовой информации о них и сохранения избранных пользователей в локальный JSON-файл.  
Приложение демонстрирует работу с GUI, внешним API, валидацией пользовательского ввода и простым постоянным хранилищем данных в формате JSON.

---

## Как используется GitHub API

Приложение обращается к публичному REST API GitHub по эндпоинту:

```text
GET https://api.github.com/users/{username}
```

где `{username}` — введённый логин пользователя GitHub.  
Из ответа используются поля: `login` (логин пользователя), `html_url` (ссылка на профиль), `public_repos` (количество публичных репозиториев), `followers` (количество подписчиков). Полученные данные отображаются в интерфейсе и при необходимости сохраняются в список избранных.

## Требования для запуска

- Python 3.10 или выше  
- Установленная библиотека `requests`:

```bash
python -m pip install requests
```

Файлы проекта:

- `main.py` — основной файл приложения  
- `favorites.json` — файл с избранными пользователями (создаётся и обновляется автоматически)  
- `README.md` — документация к проекту

## Как запустить

1. Клонируйте репозиторий или скачайте файлы проекта.  
2. Установите зависимости (при необходимости):

   ```bash
   python -m pip install requests
   ```

3. Запустите приложение:

   ```bash
   python main.py
   ```

После запуска откроется окно приложения GitHub User Finder.

---

## Как пользоваться

1. Введите логин пользователя GitHub в поле «Логин пользователя GitHub».  
2. Нажмите кнопку «Найти».  
   - Если пользователь найден, он появится в списке «Результаты поиска» с краткой информацией (логин, количество репозиториев и подписчиков).  
   - Если логин неверный или пользователь не существует, появится сообщение о том, что пользователь не найден.  
   - Если поле поиска пустое, приложение покажет предупреждение, и запрос к API выполнен не будет.  
3. Выделите нужного пользователя в списке результатов и нажмите «Добавить в избранное».  
4. Список избранных пользователей отображается в нижней части окна и сохраняется в файле `favorites.json`.  
5. При следующем запуске приложения избранные пользователи загружаются из `favorites.json` автоматически.

## Примеры использования

**Позитивный сценарий**

- Ввод: `octocat`  
- Результат: в списке появляется строка вида  
  `octocat | репозиториев: N | подписчиков: M`,  
  пользователь успешно добавляется в избранное.

**Негативный сценарий: пустой ввод**

- Ввод: пустое поле  
- Результат: приложение выводит предупреждение «Поле поиска не должно быть пустым», запрос к GitHub API не выполняется.

**Сценарий «пользователь не найден»**

- Ввод: `user_that_does_not_exist_123`  
- Результат: всплывающее сообщение «Пользователь 'user_that_does_not_exist_123' не найден.»

## Тестирование

| Тип теста   | Входные данные                    | Ожидаемый результат                                  |
|------------|------------------------------------|------------------------------------------------------|
| Позитивный | Логин существующего пользователя  | Пользователь найден, отображён в списке              |
| Негативный | Пустое поле поиска                | Показано предупреждение, запрос к API не выполняется |
| Негативный | Несуществующий логин              | Сообщение «Пользователь не найден»                   |
| Граничный  | Очень короткий логин (1 символ)   | Корректный ответ API (или «не найден»)               |
